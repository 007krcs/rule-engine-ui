# I18n

RuleFlow schemas use i18n keys (`labelKey`, `placeholderKey`, `helperTextKey`, `ariaLabelKey`) so UI text resolves at runtime.

![Builder props panel](/docs/builder-props-panel.png)

## Step-by-step integration

1. Add i18n keys in UI schema components.
2. Provide locale bundles (platform + tenant override bundles).
3. Build an `I18nProvider`.
4. Pass the provider into renderer hosts.

### 1) Add keys in schema

```json
{
  "id": "customerName",
  "type": "input",
  "adapterHint": "material.input",
  "i18n": {
    "labelKey": "customer.name.label",
    "placeholderKey": "customer.name.placeholder",
    "helperTextKey": "customer.name.helper",
    "ariaLabelKey": "customer.name.aria"
  }
}
```

### 2) Provide bundles

Store bundles in your system or ship them with the host app. RuleFlow demo merges platform bundles with tenant bundles.

### 3) Implement provider

```ts
const provider: I18nProvider = {
  locale: 'en-US',
  direction: 'ltr',
  t: (key, params) => customTranslate(key, params),
  has: (key) => customHasKey(key),
  formatNumber: (value, options) => new Intl.NumberFormat('en-US', options).format(value),
  formatDate: (value, options) => new Intl.DateTimeFormat('en-US', options).format(value),
};
```

### 4) Pass provider to renderer

```tsx
<RenderPage uiSchema={uiSchema} data={data} context={context} i18n={provider} />
```

## Bundle precedence

1. Tenant override bundle
2. Platform bundle
3. Fallback text (dev key-as-text or prod fallback)

## RTL

Renderer applies `dir="rtl"` when locale is RTL (`ar`, `he`, `fa`, `ur`).

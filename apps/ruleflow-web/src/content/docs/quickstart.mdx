# Quickstart

RuleFlow lets you ship configuration-driven UI, flows, and rules without embedding business logic in your host apps.

## In-product quickstart (no code)

If you're new, use the in-product onboarding to go from zero to a full runtime trace in a few minutes.

1. Open **Get Started** in the header (or go to [Samples](/samples)).
2. **Clone a sample config**. This creates a **DRAFT config version** you can safely edit.
3. Open **Builder** and **drag a component** onto the canvas, then click **Save**.
4. Open **Rules Builder** at `/builder/rules?versionId=...`, click **Add starter rule**, then **Save**.
5. Open **Playground**, hit **Submit**, then inspect **Trace** and toggle **Explain** mode.

## Install

```bash
pnpm install
```

## Run the product web app

```bash
pnpm dev
```

## Render a UI schema

```tsx
import { RenderPage } from '@platform/react-renderer';
import { registerMaterialAdapters } from '@platform/react-material-adapter';

registerMaterialAdapters();

<RenderPage uiSchema={uiSchema} data={data} context={context} i18n={i18n} />
```

## Execute a flow

```ts
const result = await executeStep({
  flow,
  uiSchemasById,
  rules,
  apiMappingsById,
  stateId: 'review',
  event: 'submit',
  context,
  data,
  fetchFn,
});
```

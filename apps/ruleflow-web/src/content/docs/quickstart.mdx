# Quickstart

RuleFlow lets you ship configuration-driven UI, flow transitions, and rules without embedding business logic in host apps.

![Docs overview](/docs/docs-overview.png)

## Start with the guided overview

If you are new, begin at:

- `/docs/getting-started`

That page explains `configId`, `versionId`, lifecycle states, and the full authoring workflow.

## In-product quickstart (no code)

1. Open **Get Started** (header) or `/samples`.
2. Create a draft config (or clone a sample) to get a `versionId`.
3. Open Builder, drag a component to canvas, then click **Save**.
4. Open Rules Builder, click **Add starter rule**, then click **Save**.
5. Open Playground, change a bound input, click **Submit**, and inspect **Trace** with **Explain** enabled.
6. Open Console and run governance lifecycle actions (`Submit`, `Approve`, `Promote`).

## Install

```bash
pnpm install
```

## Run the web app

```bash
pnpm dev
```

## Render a UI schema

```tsx
import { RenderPage } from '@platform/react-renderer';
import { registerMaterialAdapters } from '@platform/react-material-adapter';

registerMaterialAdapters();

<RenderPage uiSchema={uiSchema} data={data} context={context} i18n={i18n} />
```

## Execute a flow step

```ts
const result = await executeStep({
  flow,
  uiSchemasById,
  rules,
  apiMappingsById,
  stateId: 'review',
  event: 'submit',
  context,
  data,
  fetchFn,
});
```

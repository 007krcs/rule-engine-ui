# Tutorial: Component Registry

The Component Registry keeps RuleFlow pluggable:
- the runtime is headless
- UI is rendered by adapters resolved via `adapterHint`
- Builder learns how to edit props from `propsSchema`

## What a component definition looks like

```json
{
  "schemaVersion": 1,
  "components": [
    {
      "adapterHint": "company.currencyInput",
      "displayName": "Currency Input",
      "category": "Company",
      "propsSchema": {
        "type": "object",
        "properties": {
          "label": { "type": "string", "title": "Label" },
          "currency": { "type": "string", "enum": ["USD", "EUR"] }
        }
      },
      "defaultProps": { "label": "Amount", "currency": "USD" },
      "bindings": { "data": ["value"] }
    }
  ]
}
```

## Step-by-step (5 minutes)

1. **Open the registry page**
   - Navigate to `/component-registry`.

2. **Choose scope**
   - **Global** applies to all tenants in the demo.
   - **Tenant** overrides global definitions for a specific tenantId.

3. **Paste a manifest**
   - Paste JSON into **Manifest JSON**.
   - Fix validation issues if any are shown.

4. **Register**
   - Click **Register**.
   - Open Builder: new components appear in the palette immediately.

## Preview Data Notes

- Many built-in components ship with `defaultProps` so UI Kit preview can render realistic demos.
- These values are dummy data for preview/testing only.
- In production screens, bind real tenant/runtime data through `bindings` and component props.

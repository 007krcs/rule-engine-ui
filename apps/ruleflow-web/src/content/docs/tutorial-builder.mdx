# Tutorial: Create UI Upfront (Builder)

Builder is a schema authoring surface for `UISchema`:
- palette is driven by the **Component Registry**
- property panel is generated from **JSON Schema**
- validation gates block Save until issues are fixed

## Goal

Create a UI schema that renders correctly, is accessible, and is ready to run in Playground.

## Step-by-step (15 minutes)

1. **Open Builder**
   - Use **Get Started ? Step 2**, or open `/builder?versionId=...`.

2. **Drag/drop components**
   - Drag a component from the palette onto the canvas.
   - Drag canvas items to reorder or use **Move up** / **Move down** buttons.
   - Click a canvas item to edit it.

3. **Edit props (schema-driven)**
   - In the right panel, use the **Props** form generated from `propsSchema`.
   - Use **Edit as JSON** for advanced fields.

4. **Add bindings (optional)**
   - Bindings connect props to runtime data paths (ex: `data.orderTotal`).
   - Playground will show bound values and use them in rules.

5. **Fix validation issues**
   - If Save is disabled, open **Schema Preview** and click an issue.
   - Fix missing `ariaLabelKey`, invalid focus order, or missing i18n keys.

6. **Preview mode**
   - Click **Preview** in the header, or open `/builder?versionId=...&preview=1`.
   - Switch breakpoints (Desktop/Tablet/Mobile).

7. **Save**
   - Click **Save** to persist the DRAFT version.

8. **Optional: continue to flow/rules**
   - Flow JSON workflow: `/docs/tutorial-flow-editor`
   - Rule authoring: `/docs/tutorial-rules`

## How palette + property panel work

Builder reads component definitions from `/api/component-registry`.

Each `ComponentDefinition` includes:
- `adapterHint` (how the renderer chooses the adapter)
- `propsSchema` (JSON Schema subset used to generate form fields)
- `defaultProps` (starter values)
- `bindings` (keys the component supports)
